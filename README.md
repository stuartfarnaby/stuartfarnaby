## 🧠 How I Think

My approach to software engineering is guided by a few core principles:

*   **Keep it Simple:** I strive for clean, maintainable, and straightforward solutions.
*   **Tooling Follows the Project:** I believe in picking the right language or framework for the project, not bending a project to fit a preferred technology stack.
*   **User Problem First:** I always take the time to understand the user's actual problem before diving into the technical aspects of a solution. 
*   **Pragmatic PM:** I believe specification and project management should exist to make a project easier and more transparent, never to serve as a bureaucratic chore.
